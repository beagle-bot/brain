# 第一版本构想
brain——**把外部信息流，逐渐训练成一个懂你的个人认知系统**
1. 背景：ai更新速度太快，出于fomo情绪以及期望跟上热点的诉求，我会希望有地方能提供足够高质量的信息方便我阅读和积累，但实践中存在几个问题
	1. 内容太多，以[卡兹克整理的aihot为例](https://aihot.virxact.com/items/cmqqrfwu40cpwslp5zbjpmubs)，作为一个产品，其中太多纯技术向的内容我觉得无用——或者可以说，这仍然是一个编辑队列，**无法实现对我的有效推荐**
	2. 记不住，我希望看到很多信息，但是我更希望它能为我所用，但是现在可能只能用截图或者**收藏的方式简单follow，无法沉淀
	3. 何时去用，一个永恒的命题，内容宝贵，但是何时有价值？
		1. 搜索，but只有搜索吗？
		2. 怎么能够成为一种仿佛掌握在大脑里的知识，产生近乎无限的价值？
2. 核心点
	1. 在大众信息里捕捉**对我有用的信息**
	2. **由我来选择记住哪些内容**
	3. 让这些内容在未来有价值
3. 思路
	1. 明确什么东西对我有用，拆解hot的内容并输出
		1. 理想态下，定义清楚影响维度，不同人可以选择不同的特质得到自己想要的内容
	2. 内容输出，按照有价值降序排，做推荐流，左滑记忆，右滑消失
	3. 内容沉淀，一开始的构想是沉淀成markdown放在obsidian里，用rag的方式给AI记忆，但是担心不够好用，**这里如何利用需要再想想**
	4. 更长远的思考
		1. 大脑可以被分区，mvp是AI认知，后续也可以做论文、生活常识，everything
4. 我觉得目前的难点：
	1. 一是衡量“我的”相关性的维度和标签是什么，我期望能做到千人千面，或者说不同职业，不同xx的人，通过选择可以自然的明确诉求，然后按照他们的需求排序（并非推荐系统，可能只是某个规则）

> 我们最初的梦想是打造一个能够整合所有知识的统一界面——包括笔记、邮件和搜索记录等。但事实证明，手动实现各种数据整合是一项耗时费力的工作。尤其是当各大公司都推出了类似的产品时，如何在竞争中保持优势就变得更加困难了。
3.[llmwiki,AI知识库](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
>这就是关键区别：维基是一种持续更新、不断完善的知识体系。其中的交叉引用早已存在，各种矛盾之处也早已被标记出来。所有的内容都反映了你读过的所有信息。每当你添加新的资料或提出新的问题，维基的内容就会变得更加丰富。
>`原始资料——也就是您精心整理的各种原始文档。包括文章、论文、图片和数据文件等。这些资料是不可更改的：大语言模型只会读取这些资料，而绝不会对它们进行修改。这些资料就是您获取真实信息的来源。
>`wiki——这个“维基”其实是一个由大语言模型生成的 Markdown 文件目录。其中包含各种摘要、实体信息页、概念介绍、各种对比分析以及整体概览。这一整个体系都由大语言模型来管理：它负责创建页面内容，当有新信息出现时及时更新页面，同时维护各种页面之间的关联，确保所有内容的完整性。你负责阅读这些内容，而大语言模型则负责将其编写出来。
>`模式文档——其实就是一份文档（比如 Claude Code 的 CLAUDE.md，Codex 的 AGENTS.md）。这份文档向大语言模型说明了维基的结构、各种规则，以及在处理各种任务时应该遵循的工作流程。这是一份至关重要的配置文件——它决定了大语言模型是会像一个有条理的维基维护者那样行事，还是仅仅作为一个普通的聊天机器人而已。随着时间的推移，你和大语言模型会共同完善这份文档，从而找到最适合你所在领域的运作方式。
